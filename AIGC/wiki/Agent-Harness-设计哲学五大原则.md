# Agent Harness 设计哲学五大原则

## 定义

从 Claude Code 架构中提炼的五个贯穿始终的设计原则，相互支撑构成架构决策网络。每个原则都回答了一个核心的"为什么"：为什么不选更简单的方案？如果不这样设计会怎样？

## 核心观点

- 五大原则不是孤立的设计点，而是相互支撑的架构决策网络（来源：raw/御舆：解码 Agent Harness/article.md）
- 每一个"为什么"的答案都可以追溯到这五个原则之一
- 后续各章节都是这些原则在具体子系统中的展开

## 详细内容

### 原则一：异步流式优先（Async Generator First）

对话循环建立在 `AsyncGenerator` 之上，不是随意选择而是深刻的架构决策。

AsyncGenerator 同时满足三个需求：
- **增量输出**：通过 `yield` 逐步产出流式事件，上层代码实时渲染
- **可中断性**：调用者通过 `generator.return()` 随时终止
- **背压控制**：消费者处理速度跟不上时，生成器自动暂停，避免内存溢出

其他方案的短板：
- **回调模式**：随事件类型增多陷入"回调地狱"
- **Promise 链**：失去中途取消能力，用户按 Ctrl+C 时无法优雅停止
- **EventEmitter**：内存泄漏风险（忘记移除监听器）+ 类型安全问题（事件名是字符串）

> AsyncGenerator 是唯一同时满足"流式输出"、"可取消"和"类型安全"三个需求的方案。

### 原则二：安全边界内嵌（Security at the Perimeter）

权限系统不是附加的安全层，而是被内嵌到架构核心管线中。工具调用从 LLM 提出到最终执行，需经过四个安全检查点：

1. **工具可见性过滤**：在发送给 LLM 之前，过滤掉被禁止的工具——模型甚至无法"看到"不该使用的工具
2. **输入校验**：`validateInput` 在权限检查之前执行，拒绝格式不合法的参数
3. **权限决策**：`canUseTool` 综合考量权限模式、工具危险等级、用户历史决策
4. **运行时防护**：沙箱限制、超时控制、输出大小限制

设计哲学：**纵深防御（Defense in Depth）**——没有单一检查点是银弹，但每一层都可以独立短路。

### 原则三：缓存感知设计（Cache-Aware Architecture）

在 LLM API 计价模型中，Prompt 缓存可以显著降低成本和延迟。三个层面的缓存友好设计：

- **系统 Prompt 稳定性**：工具列表不变时 Prompt 字节内容保持一致，命中 API 侧缓存
- **子智能体的缓存共享**：Fork 模式下子智能体继承父智能体的 `renderedSystemPrompt`，保证缓存命中率
- **消息历史的不可变性**：已发送给 API 的消息只有追加操作，保证缓存键的稳定性

### 原则四：渐进式能力扩展（Progressive Capability）

四级扩展模型，从内建到外部、从简单到复杂：

| 扩展级别 | 机制 | 适用场景 | 扩展者角色 |
|---------|------|---------|----------|
| **工具（Tool）** | 实现 `Tool` 类型接口 | 添加原子操作能力 | 核心开发者 |
| **技能（Skill）** | Markdown + 脚本的声明式工具 | 封装可复用任务模板 | 高级用户 |
| **插件（Plugin）** | 带生命周期的工具包 | 组织相关工具和配置 | 生态开发者 |
| **MCP 服务器** | 标准化协议的外部工具集成 | 第三方工具生态 | 第三方开发者 |

设计哲学：**渐进增强**——每一级都建立在前一级基础之上，而非替代它。

### 原则五：不可变状态流转（Immutable State Flow）

状态管理借鉴 Redux/Zustand 的不可变状态模式：

- **Updater 函数模式**：状态更新接收 `(prev: T) => T` 函数，确保每次更新基于前一个状态，避免竞态条件
- **引用相等性检查**：通过引用比较确保只有真正变化时才触发通知，避免不必要的重渲染
- **订阅/取消订阅模式**：返回清理函数，防止内存泄漏

对话循环层面：跨迭代状态通过整体替换更新，每次迭代解构不可变快照。

好处：状态变化可预测、可追溯；子智能体安全接收状态快照而不会意外修改；推测执行场景下状态回滚简单安全。

## 原则间的关联

```
异步流式优先 → 不可变状态流转
异步流式优先 → 缓存感知设计
不可变状态流转 → 安全边界内嵌
安全边界内嵌 → 缓存感知设计
不可变状态流转 → 渐进式能力扩展
```
