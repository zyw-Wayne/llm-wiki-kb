# Context 压缩

## 定义

Context 压缩（Compaction）是 Anthropic 提出的长程任务上下文管理策略：在上下文接近窗口限制时，将对话内容"有损压缩"，抛弃冗余历史信息，开启新的上下文窗口，仅保留核心决策与实现细节。

## 核心观点

- 最直接的思路：让模型触发"总结"动作，提炼历史对话
- 适合**多轮对话交互**类任务（来源：[[有效的-Context-工程-见知录004]]）
- Claude Code 实践：保留开发架构决策、未解决的错误和关键实现细节，同时丢弃冗余工具输出

## 工程调优思路

1. 压缩 prompt 先以「**最大召回率**」为目标编写，确保提取所有相关信息
2. 再迭代提示词，逐步消除总结中的冗余内容，提升压缩精度

## 与其他策略的对比

| 策略 | 适用场景 | 特点 |
|------|----------|------|
| 压缩（Compaction） | 多轮对话交互任务 | 有损压缩，丢弃冗余历史 |
| 结构化笔记 | 持久化保存工作记忆 | 极小上下文开销，外部文件持久化 |
| 多智能体架构 | 复杂任务分解 | 分而治之，降低单 Agent 上下文压力 |

## Claude Code 四级压缩实现（来源：[御舆：解码 Agent Harness](../raw/御舆：解码%20Agent%20Harness/article.md)）

Claude Code 的压缩实现远比"总结对话"更精细，分为四个递进层级：

| 级别 | 名称 | 成本 | 触发条件 |
|------|------|------|---------|
| L1 | Snip（裁剪） | ≈0 | 用户手动标记旧工具结果 |
| L2 | MicroCompact（微压缩） | 极低 | 服务端缓存过期（时间触发） |
| L3 | Collapse（折叠） | 中等 | 上下文利用率达 90%（主动） |
| L4 | AutoCompact（自动压缩） | 高 | 令牌超阈值（被动兜底） |

**双阶段提示结构**：压缩提示要求 LLM 先输出 `<analysis>` 思维草稿本（事后丢弃），再输出 `<summary>` 正式摘要（进入上下文）——思考不计费，结果才进入上下文。

**断路器保护**：连续压缩失败 3 次后停止尝试，防止 API 调用雪崩（真实数据：曾有会话出现 3,272 次连续失败）。

**最佳实践**：主动在 70% 时手动 `/compact` 并附带保留提示，比等到 95% 自动压缩效果更好。
