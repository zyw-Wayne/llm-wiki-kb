# Token 优化技巧

通过压缩 AI 回复格式，大幅削减 output token，从而降低费用、提升速度。

## 核心原理

LLM 是**自回归模型**：每生成一个 token 都要做一次完整的 forward pass。

- 生成 180 个 token → 跑 180 次推理
- 生成 45 个 token → 跑 45 次推理

**少说废话 = 少做计算 = 物理上更快。**

### 复利效应

在 Agent 多轮对话中，每一轮的 output 都会成为下一轮的 input。

- 第一轮多说 100 个废话 token
- 后面每轮都得多处理这 100 个 token
- 十轮下来 = 多处理 1000 个不必要 token

**省 output 的效果在多轮场景下是复利累积的。**

## 两种实践方案

### 穴居人模式（英文）

让 Claude 像原始人一样说话，一个词一个词往外蹦。

| 正常模式 | 穴居人模式 |
|---------|-----------|
| "我已经为您执行了网页搜索工具" (8 token) | "工具完成" (2 token) |
| 180 token / 任务 | 45 token / 任务 |

System Prompt 示例：
```
You are a caveman. Speak in grunts. Minimal words. No explanations.
```

### 文言文模式（中文）

用文言文约束回复格式，压缩率极高。

System Prompt 示例：
```
汝以文言作答。凡回复，惜字如金，不赘不饰。先行后言，果先因后。若无必要，勿增实体。
```

**核心逻辑**：文言文本身就是高压缩语言——「善」= 现代中文「好的我明白了」，「诺」=「好的我马上去办」。

## 实测数据

同一个编码任务（Python 词频统计 + 单元测试 + 调试）：

| 指标 | 正常模式 | 文言文模式 | 变化 |
|------|---------|-----------|------|
| 耗时 | 73 秒 | 30 秒 | -59% |
| 工具调用 | 6 次 | 4 次 | -33% |
| 回复长度 | 完整段落 | "三测皆通" (4字) | -97% |
| 代码质量 | 正确 | 正确 | 一致 |

**关键发现**：工具调用也减少了。正常模式多写了 2 个测试用例反而引入 bug；文言文模式精准写了 3 个，一次通过。"惜字如金"不只影响回复风格，连干活方式都变了——**更精准、更克制**。

## 会变笨吗？

**通常不会。** 原因：

1. **Output 格式 ≠ 思考深度**：Claude 在回复前有独立的 extended thinking（thinking tokens），不受 output 格式指令影响。压缩的是「演示层」，不是推理链。
2. **类比**：让教授写一句话摘要 vs 写一整页报告，教授的理解深度不变。

Anthropic 工程师 Dickson Tsai 确认：在 agent 场景下省的主要是 output token，input token（工具返回、上下文等）才是真正大头。

## Token 经济学

| Token 类型 | 价格（Claude Sonnet） |
|-----------|---------------------|
| Input | $3 / 百万 token |
| Cached Input | $0.30 / 百万 token |
| Output | $15 / 百万 token |

- Output 价格 = Input 的 **5 倍**，Cached Input 的 **50 倍**
- Output 在总 token 量中约占 4%，但账单上可能占 **30%+**
- 加上复利效应，省 output 的经济价值相当可观

## 适用性

- **不限于 Claude**：对所有自回归模型适用（GPT、Gemini 等）
- **代码不受影响**：代码本身不会被压缩，省掉的是中间过程的语气词、过渡句和客套话
- **推荐在 CLAUDE.md 中配置**，持久生效

## 与其他优化方法的关系

- 对比 [[Chain-of-Thought]]：CoT 让模型多说（思考过程显式化），文言文模式让模型少说（输出压缩）。两者不矛盾——CoT 管的是推理阶段，文言文管的是输出阶段。
- 对比 [[Context-Engineering]]：Context Engineering 管理输入端（塞什么进上下文），Token 优化管理输出端（回复多精简）。
- 对比 [[AI编程行为准则]]：都是用行为约束提升效率，但侧重点不同——行为准则约束"怎么干活"，Token 优化约束"怎么说话"。
- 对比 [[Claude-Code-系统化用法]]：系统化用法管怎么搭工作流，Token 优化管怎么省资源，两者互补。
