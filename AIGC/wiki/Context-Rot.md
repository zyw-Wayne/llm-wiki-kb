# Context Rot

## 定义

Context Rot（上下文腐烂）是指随着 LLM 上下文窗口中的 token 数量增加，模型推理能力显著下降的现象。即使未超过模型标称的最大上下文窗口，对话越长，回复质量也会越差。

## 核心观点

- **最大上下文窗口 ≠ 最佳注意力窗口**。Gemini 2.5 Pro 的 1M 上下文，在 tokens 量来到 4w 左右时质量已开始下降（来源：[[有效的-Context-工程-见知录004]]）
- 上下文需被视为**有递减收益边际的有限资源**，而非越大越好
- Context Rot 如同人类信息过载时的思维混乱——过长、充满干扰的上下文同样会降低模型推理能力

## 三大影响因素

来自 Chroma 团队《Context Rot》实验研究：

1. **Context 输入越长 → 注意力被稀释**
2. **问题与关键信息的语义相似度越低 → 模型越难匹配到答案**
3. **关键信息与周围干扰内容的语义相似度越高 → 干扰增强，模型难以分辨**

这三个因素会相互放大，导致性能显著下降。反过来说：控制 Context 长度、减少干扰项、提升问题与有效信息的相似度，就能提升 Agent 处理效果。

## Chroma 实验细节

在传统 NIAH（Needle in a Haystack）基础上拓展了 4 种测试任务：

| 测试类型 | 核心变量 | 发现 |
|----------|----------|------|
| 针-问题对相似度 | 语义理解难度 vs context 长度 | 相似度越低，受长度影响越大；短 context 时模型对低相似度问题处理成功率更高 |
| 干扰项测试 | 干扰数量 + 位置 vs context 长度 | context 越长，干扰项影响加剧 |
| 针-干草堆相似度 | 主题相关度 vs context 长度 | 主题越相关越难识别；格格不入的信息反而更容易被发现（类似找茬游戏） |
| 段落结构测试 | 逻辑连贯 vs 杂乱颠倒 | 结构连贯性影响模型推理表现 |

### NIAH 拓展方向

传统 NIAH 考察直接词汇匹配的"大海捞针"能力，但实际场景更复杂：
- 需要处理"针-问题"间的模糊语义（如"喂了长颈鹿"→"动物园里有什么动物"）
- 真实上下文充满干扰项（如"大学教授的建议"干扰"大学同学的建议"）

## 实践启示

- 无论多简单的任务，context 长度增加都会导致能力下降
- 测试 AI 产品时，需找到较佳的 context 长度区间
- 针在语义上与周围"格格不入"反而更容易被模型识别——这一发现对信息组织有启发

## 症状表现

- **回答深度降低**：难以结合前文细节，需要重新发送关键 Prompt
- **混乱归因**：将不相关细节关联起来，得出奇怪结论
- **忘记前序指令**：丢失早期对话中的约束要求
