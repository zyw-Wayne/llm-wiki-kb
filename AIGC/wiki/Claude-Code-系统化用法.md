# Claude Code 系统化用法

## 定义

从"跟 AI 聊天"到"调用系统"的 12 种进阶用法，核心转变：不再每次从零开始写提示词，而是通过配置、钩子、规则文件将 AI 行为固化为可复用的工程系统。

## 核心论点

普通人和高手的差距不在提示词能力，而在使用姿势：

> 每一次，你都在从零开始。你的项目背景、代码规范、历史决策——下一次对话全部清零。

高手做的第一件事是解决这个根本问题：让 AI 不再从零开始。

## 12 种用法

### 用法 01：从「写代码」→「跑流程」

输入 `/tdd`，AI 自动：写测试 → 写实现 → 跑验证 → 修复。你要的是结果，高手启动的是流程。

### 用法 02：从「每次解释」→「一次定义」

将项目规则写入 `~/.claude/rules/`，每次对话自动携带。把「提示词」变成「系统配置」。

### 用法 03：从「短期记忆」→「长期记忆」

开启 memory 持久化钩子，会话结束自动把关键决策、踩过的坑、项目状态写进记忆文件。下次 AI 开口就知道项目阶段。

### 用法 04：从「单线程」→「多代理协作」

拆任务，多个子代理并行执行。主代理规划，代码审查、安全扫描、测试验证各由专属代理处理。从「工具」升级成「团队」。

### 用法 05：从「写提示」→「写指令」

`/review`、`/refactor`、`/analyze` 等斜杠命令背后是完整执行协议——检查维度、输出格式、优先级全部预定义。

### 用法 06：从「随机输出」→「可控输出」

通过 rules 约束 AI 行为边界：「不允许直接删文件」「修改前必须先写测试」。AI 在你划定的轨道上运行。

### 用法 07：从「写完就算」→「自动验证」

post-tool hook 在每次代码写入后自动触发：运行测试 → 检查覆盖率 → 不通过则打回重写。

### 用法 08：从「一个工具」→「统一系统」

一套 ECC 规则跑遍多个平台（Claude / Cursor / Codex），不依赖某一个工具，掌控整个 AI 工作流。

### 用法 09：从「人工判断」→「自动安全」

AgentShield：102 条安全规则，覆盖 SQL 注入、命令注入、路径穿越等。pre-tool hook 在执行前拦截危险操作。

### 用法 10：从「任务执行」→「任务编排」

定义 workflow：接收需求 → 拆解任务 → 分配子代理 → 汇总结果，整条流水线一次触发。

### 用法 11：从「临时使用」→「长期积累」

每解决一类问题就抽象成一个 skill 文件。个人 skill 库越积越厚，AI 越用越聪明。经验从「消耗品」变成「资产」。

### 用法 12：从「工具思维」→「系统思维」（最关键）

前 11 条是具体技法，这条是底层认知。你在「用工具」，他在「搭系统」。思维模式不同，产出可以差 10 倍。

## 关键洞察

- 模型差距 ≠ 生产力差距：用 GPT-4o 搭了一套系统的人，产出可以远超用 Claude Opus「聊天」的人
- 两种工作方式对比：临时调用（即用即弃、不可复用）vs 系统配置（持续复用、自动执行、越用越聪明）
- 不是你不会写提示词，而是你一直在用错误的姿势用 AI

## 底层机制深度对照（来源：[御舆：解码 Agent Harness](../raw/御舆：解码%20Agent%20Harness/article.md)）

12 种用法对应的 Claude Code 底层工程实现（Ch5-8 源码解析）：

| 用法 | 底层机制 |
|------|---------|
| 用法 02：一次定义规则 | 六层配置系统：userSettings/projectSettings 分层存储，深度合并规则 |
| 用法 03：长期记忆 | Agent 记忆系统：四类型（user/feedback/project/reference），Fork 模式后台提取 |
| 用法 06：可控输出 | 配置系统安全边界：projectSettings 排除、allowManagedHooksOnly、pluginOnlyPolicy |
| 用法 07：自动验证 | 钩子系统 PostToolUse/Stop 事件：结构化 JSON 响应协议，asyncRewake 唤醒模型 |
| 用法 09：自动安全 | 钩子系统 PreToolUse 事件：matcher 精确过滤，退出码 2 阻止并通知模型 |

**关键工程洞察（来自 Claude Code 源码）**：
- "记忆越多越好"是误区：MEMORY.md 有 200 行/25KB 双重上限，过多记忆会干扰模型判断
- 上下文管理有四级压缩（Snip/MicroCompact/Collapse/AutoCompact），主动 70% 时压缩比等待 95% 自动触发效果更好
- 钩子系统存在三层安全门禁，克隆的仓库中的 hooks 需要工作区信任确认才能执行
