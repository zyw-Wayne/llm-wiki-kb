# Prompt 工程方法论全景

## 定义

一套系统化的提示词优化框架，涵盖 7 种经过验证的技巧，分别解决 LLM 使用中的四大痛点：抢答（浮于表面）、过载（焦点分散）、模糊（自由发挥）、不自检（第一版即终稿）。核心理念：**大多数 AI 输出问题不在模型，在提问方式**。

## AI 翻车的四大根因

- **模型太急**：LLM 本质是预测下一个词，天然倾向抢答，复杂问题也想秒回 → 答案浮于表面
- **信息过载**：提示词越长越好是误区，任务/上下文/示例/格式全塞进去 → 模型焦点全散
- **表述模糊**：问题结构不清晰 → 模型自由发挥，技术上没错但完全不是你想要的
- **从不自检**：LLM 几乎不会主动检查自己的回答，第一版输出基本就是草稿

## 技巧详解

### 1. 推理框架：Chain of Thought + Tree of Thought

**Chain of Thought（思维链）**：让模型一步步解释推理过程，不要直接跳到结论。

最简单的触发词：
```
"让我们一步一步地思考"
```

**Tree of Thought（思维树）**：CoT 升级版，不只走一条路，同时探索多个方案再选最优。
```
"提出3种不同的方法来解决这个任务。
对每种方法列出优缺点。
然后选择最好的那个并展开详细说明"
```

> 与现有 CoT 页面的关联：[[Chain-of-Thought]] 已有详细 Claude Code 实战用法（架构决策/Bug诊断/边界推演/方案质疑），本文补充了 ToT 视角和通用触发方式

### 2. Meta Prompting（元提示）

不急着让 AI 干活，先让它帮你把问题问好。模型会给你一个精炼过的提示词版本，拿优化版再跑，结果好一大截。

```
"这是我的请求：[你的提示词]

改进这个提示词，使其能产出最准确、最有条理的回答。
补充缺失的上下文，明确任务并消除歧义"
```

### 3. Step-Back Prompting（退后一步提示）

不知道该问什么具体问题时，先退一步问宏观问题，拿到框架后再精准出击。

```
第1步："决定新产品营销成功的关键因素有哪些？"
第2步："根据这些因素，为[你的产品]提出一个营销策略"
```

先全局后细节，深度和结构都会好很多。

### 4. ReAct（推理+行动）

Reason + Act，让模型不要一口气冲到终点，走一步看一步。特别适合信息分析、逻辑验证、长文本处理。

```
"思考这个问题。
决定需要什么行动。
检查结果。
然后给出最终答案"
```

### 5. 上下文管理

对话越长，AI 越容易"失忆"。定期让 AI 做总结，拿到总结后作为下一轮提示词的基础，模型重新聚焦。

```
"用简短的结构化形式总结我们对话到目前为止的关键上下文"
```

### 6. Prompt Chaining（提示词链）

大任务拆成一串小任务，每一步建立在上一步基础上。

```
第1步："分析这个产品的市场"
第2步："确定主要目标受众"
第3步："根据前面的分析，提出一个营销策略"
```

### 7. Reflexion（自我反思）

LLM 第一版回答基本是草稿。让模型自己当批评家审视回答，第二版通常有质的提升。

```
"现在以严格的批评者身份重新阅读你的回答。
哪些部分是薄弱的、不准确的、或缺失的？
改进这个回答"
```

## 技巧组合建议

- **简单问题**：CoT（一句"让我们一步一步地思考"即可）
- **复杂分析**：Step-Back → CoT → Reflexion（先建框架，再推演，最后自检）
- **多步骤项目**：Prompt Chaining + 上下文管理（拆步骤 + 定期总结防遗忘）
- **高准确度要求**：ReAct + Reflexion（边查边改 + 最终自审）
