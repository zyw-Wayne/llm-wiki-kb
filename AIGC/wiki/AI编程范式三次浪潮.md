# AI 编程范式三次浪潮

## 定义

AI 辅助编程从 2021 年至今经历的三次清晰的范式演进，每次都深刻改变了开发者与代码之间的关系。当前（2025 至今）正处于第三次浪潮——自主智能体时代。

## 核心观点

- AI 编程工具的演进方向始终是"给 LLM 更多的行动能力"（来源：raw/御舆：解码 Agent Harness/article.md）
- Agent Harness 是这个演进方向在架构层面的必然产物
- LLM 的真正价值不在于生成文本，而在于作为推理引擎来编排工具调用

## 三次浪潮

### 第一次浪潮（2021-2022）：代码补全时代

**标志事件**：GitHub Copilot 诞生

**核心范式**："行内补全"——AI 基于当前文件上下文预测下一行或代码块，开发者通过 Tab 键接受。

**能力边界**：
- 看到的是光标前后几十行代码
- 只输出片段级建议
- 不会跨文件推理
- 不会理解项目结构
- 不会主动执行操作

模式特征：**高度被动、高度局部化**。

### 第二次浪潮（2023-2024）：对话式助手时代

**标志产品**：Cursor、Windsurf、Continue 等编辑器嵌入式工具

**核心范式**：自然语言描述需求，AI 跨多个文件生成代码。

**能力边界**：
- 能写代码，但**不能运行代码**
- 能建议测试，但**不能执行测试**
- 能发现问题，但**不能修复后验证**
- 开发者在 AI 和终端之间不断切换，充当人肉"胶水"

模式特征：**受限于编辑器边界，人类充当执行层**。

### 第三次浪潮（2025 至今）：自主智能体时代

**标志产品**：Claude Code、OpenAI Codex CLI 等终端原生 Agent

**核心范式**：AI 在终端中自主执行任务。

**能力**：
- 直接运行 Shell 命令
- 读写文件系统
- 执行测试套件
- 操作 Git 版本控制
- 遇到错误时自主调整策略、迭代修复

模式特征：**从 Chatbot 到 Agent 的根本性转变，LLM 从文本生成器升级为任务编排器**。

## 工具调用的突破性意义

**没有工具调用时（封闭系统）**：知识截止于训练数据，推理局限在上下文窗口，输出只能是文本。如同博学的朋友被困在没有窗户和电话的房间——知道如何诊断疾病，但无法使用听诊器。

**有了工具调用后（开放系统编排器）**：调用搜索引擎获取实时信息、调用代码解释器执行计算、调用文件系统读写代码、调用 Git 操作版本控制——每次工具调用都是 LLM 与真实世界的一次握手。

工具调用带来的六大工程挑战（催生 Agent Harness 的原因）：
1. 工具注册与发现
2. 参数校验
3. 权限管控
4. 错误恢复
5. 状态一致性
6. 并发与调度

## 简单封装 vs Agent Harness 的本质区别

**简单封装**（反模式）：
```
while (true) { callAPI(); parseResponse(); executeTool(); }
```
假设了理想化世界：API 不超时、输出永远格式正确、上下文无限、所有操作安全。

**Agent Harness**（正确路径）：系统性解决流式输出、权限管控、上下文管理、错误恢复、状态持久化、可扩展性六大工程挑战。

类比：简单封装 = 给汽车装遥控器（能前进后退，没有方向盘助力/刹车/安全气囊）；Agent Harness = 设计一整辆安全可靠的自动驾驶汽车。
